# IBOK Bank Mockup - واجهات بنك الخرطوم

## نظرة عامة

هذا المشروع يحتوي على واجهات (UI) لمحاكاة موقع بنك الخرطوم (iBOK - Internet Banking Online Khartoum). يتضمن صفحة تسجيل دخول وشاشة التحقق من البيانات (OTP).

## محتويات المشروع

### الملفات المضمنة

```
ibok_interfaces/
├── login.html          # صفحة تسجيل الدخول
├── verification.html   # شاشة التحقق (OTP)
├── styles.css          # ملف التصميم الرئيسي
├── script.js           # ملف التفاعلات والوظائف
└── README.md           # هذا الملف
```

## كيفية الاستخدام

### فتح الملفات
1. استخرج ملفات المشروع
2. افتح ملف `login.html` في متصفح الويب
3. استخدم الواجهات كما هو موضح أدناه

### اختبار صفحة تسجيل الدخول
1. أدخل رقم حساب بـ 7 أرقام (مثال: 1234567)
2. أدخل كلمة مرور تحتوي على:
   - 8 أحرف على الأقل
   - أحرف كبيرة (A-Z)
   - أحرف صغيرة (a-z)
   - رمز خاص (!@#$%^&*)
   - مثال: Password123!
3. انقر على "تسجيل الدخول"

### اختبار شاشة التحقق
1. انقر على "تسجيل الدخول" من صفحة تسجيل الدخول
2. سيتم نقلك إلى شاشة التحقق
3. أدخل 6 أرقام في حقول OTP
4. يمكنك:
   - الكتابة مباشرة والتنقل التلقائي
   - لصق الرمز كاملاً
   - استخدام أسهم لوحة المفاتيح للتنقل
5. انقر على "تأكيد التحقق"

## التحقق من صحة البيانات

### رقم الحساب
- يجب أن يكون 7 أرقام فقط
- لا يقبل أحرف أو رموز

### كلمة المرور
- 8 أحرف على الأقل
- يجب أن تحتوي على أحرف كبيرة (A-Z)
- يجب أن تحتوي على أحرف صغيرة (a-z)
- يجب أن تحتوي على رمز خاص (!@#$%^&*)

### رمز التحقق (OTP)
- 6 أرقام فقط
- يتم التنقل التلقائي بين الحقول

## التخصيص

### تغيير الألوان
عدّل متغيرات CSS في بداية ملف `styles.css`:

```css
:root {
    --primary-red: #D32F2F;      /* اللون الأساسي */
    --secondary-cyan: #00BCD4;   /* اللون الثانوي */
    /* ... */
}
```

### تغيير النصوص
عدّل النصوص مباشرة في ملفات HTML:
- غيّر عنوان الصفحة
- غيّر الرسائل والتعليمات
- أضف/احذف عناصر حسب الحاجة

### إضافة وظائف
أضف وظائف جديدة في ملف `script.js`:
- معالجة الأخطاء المخصصة
- التكامل مع API
- حفظ البيانات محلياً

## الملاحظات الأمنية

⚠️ **تنبيه أمني**: هذه واجهات تجريبية فقط. عند استخدامها في بيئة إنتاجية:

1. **لا تخزن كلمات المرور** في الكود الأمامي
2. **استخدم HTTPS** دائماً
3. **تحقق من البيانات** على الخادم (Server-side validation)
4. **استخدم حماية CSRF** (Cross-Site Request Forgery)
5. **طبق معايير الأمان** الدولية
6. **استخدم التشفير** لنقل البيانات الحساسة
